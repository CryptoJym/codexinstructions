Vibe Coding with Codex CLI

Overview Install First Run Add MCP Practice Cheatsheet

1. What You’ll Build & Why

This crash‑course shows complete beginners (a.k.a. vibe coders) how to:

  • Install Codex CLI locally in under 5 minutes.
  • Launch an interactive chat in your terminal and “talk” your way to working code.
  • Add external super‑powers (web search, GitHub, DB access) via the MCP.
  • Ship your first AI‑generated feature without hand‑coding, just by describing what you want.

Follow the purple ► Do this now prompts – they are the exact commands to paste. No prior Node, Git, or OpenAI experience required.

2. One‑Time Setup (≈ 5 min)

  1. Install Node JS ≥ 18 (skip if you already have it).
    ► Do this now – Mac & Linux:
    brew install node

  2. Install Codex CLI globally:
    ► Do this now npm install -g @openai/codex

  3. Grab your OPENAI_API_KEY from OpenAI dashboard and export it:

    # macOS / Linux / WSL
export OPENAI_API_KEY="sk-..."

    Windows PowerShell: $Env:OPENAI_API_KEY="sk-..."

  4. Create a fresh project folder and init Git:

    mkdir hello-vibe
cd hello-vibe
git init && git commit --allow-empty -m "🔰 start"

3. Your First “Talk‑to‑Code” Session

Let’s scaffold a tiny web API just by describing it.

  1. Launch Codex in Suggest mode:

    codex

  2. At the prompt, paste:

    Create an Express.js server that says "✨ Hello, Vibe Coder!" on GET /

    Codex proposes file edits & a command. Type y to approve each.

  3. Run the server (Codex will suggest): node server.js. Visit http://localhost:3000 🎉

Heads‑Up: Approval Modes

Start in Suggest ( default) so nothing runs without your OK. Switch later:
• -auto-edit = auto apply file changes
• -full-auto = run everything (sandboxed)

4. Super‑Power Up with MCP (Optional)

MCP tools let Codex reach beyond your repo. We’ll add Brave Search so Codex can look things up.

  1. Get a Brave Search API token (sign up).

  2. Run the MCP server in a new terminal:

    BRAVE_SEARCH_TOKEN=xxx npx -y @modelcontextprotocol/server-brave

  3. Edit ~/.codex/config.yaml (create if missing):

    model: o3
servers:
  - type: stdio
    command: npx
    args: ["-y", "@modelcontextprotocol/server-brave"]

  4. Restart Codex. Ask:

    Search "express rate limit middleware" and add the best package to this project

    Watch Codex call the search tool, decide a package, edit package.json, and install it.

5. Practice Missions

Try these mini‑quests to build confidence:

  • Mission 1: “Add a /users endpoint returning JSON”.
  • Mission 2: “Write a Jest test for / route and make it pass”.
  • Mission 3: “Refactor server to TypeScript”. Approve the auto‑refactor.
  • Mission 4: Turn on -auto-edit and ask Codex to apply Prettier formatting project‑wide.

Commit after each mission: git commit -am "✅ mission X done".

6. Quick Cheatsheet

Common Commands

  • codex – interactive chat
  • codex -full-auto "prompt" – one‑shot automation
  • codex --upgrade – update CLI
  • git diff – review AI edits

Prompt Starters

  • “Explain what this repo does.”
  • “Add dark‑mode CSS to the site.”
  • “Fix the failing tests.”
  • “Search web for the best caching lib and integrate it.”

7. Next Steps

🎯 Pair Codex with GUI editors like Cursor or Windsurf for the best of both worlds. Explore additional MCP servers (GitHub, PostgreSQL, Google Maps) to tailor Codex to your domain.
